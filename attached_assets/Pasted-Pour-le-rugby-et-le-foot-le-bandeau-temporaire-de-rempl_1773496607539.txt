Pour le rugby et le foot, le bandeau temporaire de remplacement sur le Display apporte une vraie valeur de démo :

il rend l’action visible pour le public

il valorise la régie live

il exploite naturellement match_events et match_substitutions

sans polluer l’affichage principal du score

Je te donne ce qu’il faut ajouter au pack Replit.

1) Règle produit à donner à Replit
Ajouter un bandeau temporaire de remplacement sur le Display pour rugby et football.

Objectif :
- lorsqu’un remplacement est validé dans l’Operator, le Display doit pouvoir afficher un bandeau temporaire
- ce bandeau ne doit pas casser la lisibilité du score principal
- il doit disparaître automatiquement après quelques secondes
- il doit être piloté par les événements match_events et/ou le broadcast live

Sports concernés en priorité :
- rugby
- football

Comportement attendu :
- affichage pendant 4 à 6 secondes
- texte lisible sur écran LED
- pas de surcharge
- ne pas bloquer le score / chrono
2) Prompt Replit — implémentation du bandeau de remplacement
Implémente un bandeau temporaire de remplacement sur le Display pour rugby et football.

Contexte :
- les remplacements sont saisis dans l’Operator
- ils sont journalisés dans match_events
- ils peuvent aussi être stockés dans match_substitutions
- le Display principal doit rester lisible

Objectif :
- quand un événement rugby_substitution ou football_substitution est émis, afficher un bandeau temporaire sur le Display
- le bandeau doit apparaître immédiatement
- il doit disparaître automatiquement après un délai court

Comportement attendu :
1. le bandeau s’affiche lors de la réception d’un événement de remplacement
2. il affiche :
   - équipe concernée
   - joueur sortant
   - joueur entrant
   - éventuellement numéro
3. durée configurable simple, par exemple 5000 ms
4. ne pas gêner le score principal
5. ne pas rester bloqué si plusieurs événements arrivent
6. si plusieurs remplacements arrivent rapidement, gérer une file simple ou remplacer le précédent proprement

Je veux :
- les fichiers complets à modifier
- la logique Operator si nécessaire pour diffuser l’événement
- la logique Display pour écouter et afficher le bandeau
- la structure de données recommandée
- un mini plan de test
3) Payload recommandé pour le bandeau

Quand l’Operator valide un remplacement, il faut diffuser un patch ou événement live clair.

Événement match_events

Type :

rugby_substitution

football_substitution

Payload :

{
  "team_side": "home",
  "player_out_id": "uuid-sortant",
  "player_out_name": "Dupont",
  "player_out_number": "12",
  "player_in_id": "uuid-entrant",
  "player_in_name": "Martin",
  "player_in_number": "18",
  "reason": "tactical",
  "is_temporary": false,
  "is_blood_substitution": false
}
Broadcast live recommandé

En plus du journal, l’Operator peut pousser immédiatement un événement UI-friendly :

{
  "live_seq": 123,
  "overlay": {
    "type": "substitution",
    "sport": "rugby",
    "team_side": "home",
    "team_name": "Racing Club",
    "player_out_name": "Dupont",
    "player_out_number": "12",
    "player_in_name": "Martin",
    "player_in_number": "18",
    "duration_ms": 5000,
    "event_id": "sub-uuid-001",
    "emitted_at": 1730000000000
  }
}

Ça évite au Display de reconstruire le bandeau à partir de plusieurs sources.

4) Stratégie technique recommandée
Côté Operator

Au moment du remplacement :

mise à jour match_players

insert dans match_substitutions

insert dans match_events

pushPatch() avec un objet overlay

Ordre recommandé :

affichage live immédiat d’abord

persistance juste après si tu cherches la latence minimale

Côté Display

Le Display doit :

écouter overlay.type === "substitution"

stocker un état local activeOverlay

afficher le bandeau

lancer un timer de disparition

5) Structure frontend recommandée côté Display
État local
type LiveOverlay =
  | {
      type: "substitution";
      sport: string;
      team_side: "home" | "away";
      team_name?: string;
      player_out_name?: string;
      player_out_number?: string;
      player_in_name?: string;
      player_in_number?: string;
      duration_ms?: number;
      event_id?: string;
      emitted_at?: number;
    }
  | null;

Dans main.tsx ou dans le composant racine du Display :

const [overlay, setOverlay] = useState<LiveOverlay>(null);

Quand un patch broadcast arrive :

si patch.overlay?.type === "substitution" → setOverlay(...)

timer setTimeout(() => setOverlay(null), duration_ms || 5000)
