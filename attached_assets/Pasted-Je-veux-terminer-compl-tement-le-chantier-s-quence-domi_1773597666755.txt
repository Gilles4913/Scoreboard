Je veux terminer complètement le chantier “séquence dominante unique + contrat live global” dans scoreDisplay V2.

Contexte :
- une architecture de contrat live global existe déjà
- une helper centrale existe déjà :
  - apps/operator/src/live/liveMutation.ts
- une séquence dominante existe déjà :
  - matchSeqRef
- le chrono est déjà bien durci :
  - ancre persistée en DB
  - reprise à froid correcte
  - last_event_seq chargé au démarrage
- Display et QR Console filtrent déjà les vieux patches via live_seq

Le problème restant :
- le code est encore hybride
- toutes les mutations live ne passent pas encore par emitLiveMutation(...)
- ControlPage.tsx contient encore beaucoup d’anciennes mutations qui utilisent directement :
  - pushPatch(...)
  - persistLiveState(...)
  - appendEvent(...)
- certains cas spéciaux comme handleSubstitution(...) restent gérés à la main

Objectif final :
TOUTES les mutations live du match doivent passer par la helper centrale emitLiveMutation(...)

Règle cible à atteindre :
1. une mutation live génère un seq unique via la séquence dominante du match
2. le patch live est diffusé avec live_seq = seq
3. la DB persiste last_event_seq = seq
4. si la mutation concerne le chrono :
   - persister clock_anchor_epoch_ms
   - persister clock_anchor_clock_ms
5. si un événement métier doit être journalisé :
   - match_events.seq = seq
6. si un overlay doit être diffusé :
   - il doit porter le même live_seq
7. plus aucune logique dispersée ne doit contourner ce contrat
