## Matrice d’affichage par sport

L’affichage public de scoreDisplay V2 repose désormais sur une matrice hiérarchisée de paramètres et de modèles.

### 1. `org_display_settings`
Cette table porte le **tronc commun d’affichage**, indépendamment du sport :
- thème
- logos
- sponsors
- lower third
- langues
- overlays globaux
- densité générale
- options transverses d’habillage

### 2. `org_sport_settings`
Cette table porte les **règles métier du sport** :
- nombre de périodes
- durée des périodes
- shot clock
- bonus
- fautes
- temps morts
- sets
- cartes
- etc.

Elle décrit ce qui relève de la logique sportive, pas du choix visuel.

### 3. `org_display_sport_profiles`
Cette table porte la **matrice d’affichage par sport au niveau organisation**.

Elle permet, pour un sport donné dans une organisation donnée, de définir :
- les blocs visuels activés
- la densité d’affichage
- la stratégie de lisibilité LED
- la position et durée des overlays
- le template d’affichage par défaut pour ce sport

Exemples :
- Rugby : score, chrono, période, cartons, remplacements, sin bin
- Basket : score, chrono, période, shot clock, bonus, fautes, possession
- Football : score, chrono, période, cartons, remplacements, temps additionnel
- Volley : score, sets, points du set, service, tie-break
- Handball : score, chrono, temps morts, 2 minutes, avertissements, disqualifications

### 4. `display_templates`
Cette table porte la **bibliothèque système des modèles d’affichage**.

Chaque template possède :
- un `code`
- un `sport`
- un `layout_mode`
- une `config_json`
- un indicateur `is_default_system`

Exemples :
- `rugby_stade`
- `rugby_expert`
- `football_stade`
- `basket_arena`
- `handball_classic`
- `volley_sets`

### 5. `team_display_settings`
Cette table permet à une équipe de choisir son **modèle d’affichage par défaut**.

Elle sert principalement à surcharger le choix par défaut de l’organisation.

Exemple :
- l’organisation rugby utilise `rugby_stade` par défaut
- une équipe particulière peut choisir `rugby_expert`

---

## Résolution finale de l’affichage

Le Display résout sa configuration finale dans cet ordre :

1. `org_display_settings`
2. `org_sport_settings`
3. `org_display_sport_profiles`
4. `display_templates.config_json`
5. `team_display_settings` (override éventuel)

La priorité de résolution du template est :

1. `team_display_settings.default_display_template_id`
2. `org_display_sport_profiles.default_display_template_id`
3. template système par défaut pour le sport

---

## Principes produit

- chaque sport possède ses propres blocs d’affichage
- chaque organisation peut définir une matrice d’affichage par sport
- chaque équipe peut disposer d’un modèle d’affichage par défaut
- le Display public reste piloté par l’URL stable par équipe (`teamSlug`)
- le mode public par token match est supprimé
