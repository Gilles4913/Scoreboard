Il faut arrêter les correctifs partiels et passer à une architecture unique :

une séquence dominante unique par match

un contrat live global total

une seule porte d’entrée pour toute mutation live

La cible doit être :

Toute mutation live du match passe par une helper centrale unique.
Cette helper génère la séquence dominante, diffuse le patch live, persiste l’état DB, persiste last_event_seq, journalise l’événement si nécessaire, et persiste l’ancre si la mutation concerne le chrono.

Architecture cible
1) Séquence dominante unique

Il ne doit plus y avoir de concurrence conceptuelle entre :

liveSeqRef

eventSeqRef

matches.last_event_seq

La règle devient :

matchSeqRef = seule séquence dominante côté client

matches.last_event_seq = reflet persistant de cette séquence dominante

match_events.seq = même séquence quand un événement est journalisé

Donc :

plus de double logique eventSeqRef vs liveSeqRef

une seule vérité de séquence

2) Contrat live global total

Chaque mutation live doit passer par le même pipeline :

seq = nextMatchSeq()

construire le livePatch

diffuser immédiatement le livePatch

persister le dbPatch

forcer dbPatch.last_event_seq = seq

si mutation chrono :

persister clock_anchor_epoch_ms

persister clock_anchor_clock_ms

si événement :

écrire match_events.seq = seq

si overlay :

l’embarquer dans le patch live

toutes les surfaces réceptrices filtrent :

rejeter seq < lastAppliedSeq
