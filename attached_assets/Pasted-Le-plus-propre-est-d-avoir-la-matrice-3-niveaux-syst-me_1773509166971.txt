Le plus propre est d’avoir la matrice à 3 niveaux :

système : bibliothèque de templates (display_templates)

organisation + sport : matrice d’affichage par sport pour cette organisation

équipe : modèle par défaut choisi pour cette équipe

Je te recommande donc d’ajouter une table org_display_sport_profiles pour porter la matrice au niveau organisation.

Ce que cette structure permet
Niveau organisation

org_display_sport_profiles

définit la matrice d’affichage par sport pour l’organisation

porte les blocs activés

porte les choix de lisibilité LED

porte le template par défaut d’organisation pour ce sport

Niveau équipe

team_display_settings

permet de surcharger le modèle par défaut

exemple : une équipe rugby peut utiliser rugby_expert au lieu de rugby_stade

Niveau système

display_templates

contient la bibliothèque de layouts disponibles

Requête utile pour charger la config finale Display

Le Display devra à terme résoudre dans cet ordre :

org principal

sport de l’org

org_display_sport_profiles

team_display_settings

display_templates

Priorité recommandée

template équipe si défini

sinon template org_display_sport_profiles

sinon template système par défaut

Exemple de lecture SQL logique
select
  t.id as team_id,
  o.id as org_id,
  o.sport,
  p.*,
  coalesce(td.default_display_template_id, p.default_display_template_id) as resolved_template_id
from public.teams t
join public.orgs o
  on o.id = t.org_id
left join public.org_display_sport_profiles p
  on p.org_id = o.id
 and p.sport = o.sport
left join public.team_display_settings td
  on td.team_id = t.id
where t.id = :team_id;
