1) Migration SQL
begin;

alter table public.org_display_settings
  add column if not exists show_live_badge boolean not null default false;

alter table public.org_display_sport_profiles
  add column if not exists show_live_badge boolean null;

update public.org_display_sport_profiles
set show_live_badge = false
where sport in ('rugby', 'football')
  and show_live_badge is null;

commit;

Ça te donne :

un défaut global

un override par sport

2) Règle de résolution backend

Dans get-display-context, dans l’objet final résolu, ajoute :

show_live_badge:
  typeof resolvedTemplateConfig?.show_live_badge === "boolean"
    ? resolvedTemplateConfig.show_live_badge
    : typeof displayProfile?.show_live_badge === "boolean"
    ? displayProfile.show_live_badge
    : displaySettings?.show_live_badge ?? false,

Donc :

template d’abord

profil sport ensuite

org_display_settings en fallback

3) Champ à exposer au frontend

Dans le contexte Display final, ajoute :

show_live_badge: resolvedConfig.show_live_badge ?? false,
4) Côté Display / Scoreboard

Le rendu doit devenir :

{context.show_live_badge && context.status === "live" ? (
  <div style={styles.liveBadge}>LIVE</div>

ou si tu veux aussi l’utiliser pour paused :

{context.show_live_badge && ["live", "paused"].includes(context.status) ? (
  <div style={styles.liveBadge}>
